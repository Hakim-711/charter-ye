# مخطط المنصة المحاسبية الداخلية

## الهدف
تحويل المنصة من موقع تعريفي إلى نظام أعمال متكامل يقود العمليات والمحاسبة.

## المكونات الأساسية
1. `Accounting Core`
- دليل الحسابات.
- القيود اليومية.
- دفتر الأستاذ.
- ميزان المراجعة.

2. `Sales & Invoicing`
- عروض أسعار.
- فواتير مبيعات.
- متابعة التحصيل والمديونية.

3. `Purchasing & Payables`
- طلبات شراء.
- أوامر شراء.
- فواتير موردين.
- متابعة المستحقات.

4. `Operations & Contracts`
- إدارة العقود والمشاريع.
- ربط المصروفات والإيرادات بالعقد.
- متابعة التنفيذ والتسليم.

5. `Security & Audit`
- صلاحيات حسب الدور.
- سجل تدقيق كامل لكل عملية.
- نسخ احتياطي وإجراءات استرجاع.

## مبدأ الدمج
- يظل الموقع العام هو الواجهة التعريفية للعملاء.
- يتم ربط اللوحة الداخلية عبر نطاق فرعي أو مسار آمن مثل `/portal`.
- لا تظهر البيانات المحاسبية للعامة.

## اقتراح تقني مبدئي
- Frontend: Flutter Web.
- Backend API: طبقة خدمات منفصلة.
- Database: PostgreSQL.
- Auth: JWT + صلاحيات Role-based.
